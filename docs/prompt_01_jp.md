# Project: Timeblend（ブラウザ型タイムラプス生成アプリ）

## 概要
カメラ映像をブラウザ上でリアルタイム処理し、滑らかなタイムラプス動画を生成するアプリ。
フレームを間引くのではなく、複数フレームの時間平均を取ることで滑らかさを実現する。

## コアコンセプト
従来のタイムラプス：
- フレーム間引き → カクつく

本アプリ：
- 時間平均 → 滑らか

## 処理パイプライン
カメラ入力（getUserMedia）
→ WebGL2にフレーム転送（テクスチャ化）
→ GPUでフレーム加算（時間方向に蓄積）
→ 一定時間ごとに平均フレーム生成
→ Canvasに描画
→ WebCodecsでエンコード（H.264）
→ mp4-muxerでMP4化
→ 動画として出力

## 技術スタック
- WebGL2（GPUによる高速処理）
- WebCodecs（動画エンコード）
- mp4-muxer（MP4コンテナ生成）
- MediaDevices API（カメラ取得）

## 主な機能
- リアルタイムタイムラプス生成
- 平均時間（1〜60秒）指定
- 出力解像度選択
  - 720p（1280x720）
  - 1080p（1920x1080）
  - 4K（3840x2160）
- 縦横自動対応（portrait / landscape）
- MP4ダウンロード

## 重要な設計ポイント

### GPUによる時間平均
- CPUではなくWebGLで処理
- フレームをテクスチャとして蓄積

### ピンポンバッファ
- accumulation textureを2枚用意（A/B）
- フレームごとに交互に使用して上書きを防ぐ

### 時間窓ベース出力
- 一定時間分のフレームをまとめて平均
- 出力頻度で時間圧縮を制御

### 固定フレームレート
- 出力動画は固定FPS（例：30fps）
- タイムラプスは出力間隔で実現

### 解像度戦略
- 入力・処理・出力は同一解像度
- 低解像度→高解像度のアップスケールは行わない

## パフォーマンス前提
- ピクセル数に比例して負荷が増加
- 4K（約830万ピクセル）は高負荷
- 長時間録画の制約：
  - 発熱
  - メモリ使用量
  - エンコード速度
